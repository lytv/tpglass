# Phase 05.3: Add Action Buttons to Transcript Detail Modal - Research

**Researched:** 2026-03-12
**Domain:** Electron desktop app (LitElement UI), IPC communication patterns
**Confidence:** HIGH

## Summary

This phase adds action buttons (Edit, Summarize, Custom Prompt) to the translation detail modal in SttView.js. The implementation requires modifying the existing modal structure in SttView.js to add 3 buttons in the header, implementing inline editing for the original text, and potentially creating new backend API endpoints for summarize/custom-prompt functionality. The project already has translation infrastructure (TranslationService) and prompt templates that can be reused.

**Primary recommendation:** Implement Edit as inline editing within existing modal, Summarize/Custom Prompt as separate modal windows using the existing LLM infrastructure via featureBridge.js IPC pattern.

### Pattern 1: Modal Header Action Buttons
**What:** Add action buttons to the modal header in SttView.js
**When to use:** When implementing the 3 action buttons (Edit, Summarize, Custom Prompt)
**Example:**
```javascript
// Add to modal-header div in SttView.js render()
<div class="modal-header">
    <span>Translation</span>
    <div class="action-buttons">
        <button class="action-btn" @click="${() => this._handleEdit()}" title="Edit">✏️</button>
        <button class="action-btn" @click="${() => this._handleSummarize()}" title="Summarize">📝</button>
        <button class="action-btn" @click="${() => this._handleCustomPrompt()}" title="Custom Prompt">⚡</button>
        <button @click="${() => this.expandedTranslation = null}">✕</button>
    </div>
</div>
```

### Pattern 2: Inline Edit Mode
**What:** Toggle between display and edit mode for transcript text
**When to use:** Edit button implementation
**Example:**
```javascript
// In SttView.js
_isEditing = false;

toggleEdit() {
    this._isEditing = !this._isEditing;
    this.requestUpdate();
}

renderEditMode() {
    return html`
        <textarea class="edit-textarea" .value="${this.expandedTranslation.text}"></textarea>
        <div class="edit-controls">
            <button @click="${this.toggleEdit}">Cancel</button>
            <button @click="${this.saveEdit}">Save</button>
        </div>
    `;
}
```

### Pattern 3: IPC Handler Pattern
**What:** Add new IPC handlers in featureBridge.js following existing translation pattern
**When to use:** Creating summarize and custom prompt endpoints
**Example:**
```javascript
// In featureBridge.js
ipcMain.handle('summarize:summarize', async (event, { text }) => {
    const summarizeService = require('../features/common/services/summarizeService');
    return await summarizeService.summarize(text);
});
```

### Anti-Patterns to Avoid
- **Don't create new modals for Edit:** CONTEXT specifies inline editing
- **Don't modify ListenView for these buttons:** This is SttView-specific functionality
- **Don't hardcode prompts for Custom Prompt:** Should use placeholder/future settings integration
- **Don't skip the Save/Cancel controls:** CONTEXT explicitly requires these for edit mode

## Common Pitfalls

### Pitfall 1: Missing Save/Cancel Controls for Edit
**What goes wrong:** Edit mode has no way to confirm or cancel changes
**Why it happens:** Forgetting to add control buttons per CONTEXT spec
**How to avoid:** Add Save/Cancel buttons that appear only in edit mode
**Warning signs:** Users stuck in edit mode or changes applied immediately

### Pitfall 2: Wrong Modal for Summarize/Custom Prompt
**What goes wrong:** Trying to put summarize/custom-prompt in same modal
**Why it happens:** CONTEXT specifies "Separate modal (new modal opens)"
**How to avoid:** Create new expandedTranslation state for each modal type
**Warning signs:** UI becomes cluttered, buttons don't fit in header

### Pitfall 3: Translation Not Available for Summarize
**What goes wrong:** Summarize fails when translation is disabled
**Why it happens:** Summarize uses both original and translated text per CONTEXT
**How to avoid:** Check if translation exists, fallback to original only if needed
**Warning signs:** Summarize returns empty or errors for untranslated content

### Pitfall 4: Forgetting Edit State Reset
**What goes wrong:** Modal closes but edit state persists
**Why it happens:** Not resetting _isEditing when modal closes
**How to avoid:** Reset all edit state in the close handler
**Warning signs:** Reopening modal shows edit controls unexpectedly
