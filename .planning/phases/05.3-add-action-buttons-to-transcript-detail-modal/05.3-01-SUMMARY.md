# Phase 05.3 Plan 01: Add Action Buttons to Transcript Detail Modal

## Summary

Added action buttons (Edit, Summarize, Custom Prompt) to the translation detail modal in SttView.js. The implementation includes inline editing for transcripts, a summarize modal with TLDR/bullet points/action items, and a placeholder custom prompt modal.

## Implementation Details

### Action Buttons
- **Edit** (✏️): Opens inline editable textarea for the original transcript text
- **Summarize** (📝): Opens new modal calling summarize API, displays TLDR/bullet points/action items
- **Custom Prompt** (⚡): Opens placeholder modal (future feature - prompts configured in Settings)

### Button Placement
- All 3 buttons placed in modal header, before the close (X) button
- Order: [Edit] [Summarize] [Custom Prompt] [X]
- Styling: 14px emoji icons, transparent background, subtle hover states

### Edit Mode
- Inline textarea with dark theme styling (min-height 100px)
- Save/Cancel controls below textarea
- Save updates the message in sttMessages array and dispatches 'stt-messages-updated' event
- Cancel reverts to original text

### Summarize Modal
- Separate modal with loading state
- Uses both original and translated text as input
- Calls window.api.summarize.summarize() via IPC
- Fallback placeholder if API not available

### Custom Prompt Modal
- Placeholder message: "Configure prompts in Settings"
- Ready for future implementation (prompts from Settings)
