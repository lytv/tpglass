# Phase 05.3: Add Action Buttons to Transcript Detail Modal - Context

**Gathered:** 2026-03-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Add action buttons to the translation detail modal in SttView.js. The modal appears when user clicks on a translated transcript message. Currently it only shows text with a close button.

**Note:** This phase adds Edit, Summarize, and Custom Prompt buttons - NOT Copy/Save (those already exist in ListenView).

</domain>

<decisions>
## Implementation Decisions

### Action buttons (3 buttons)
- **Edit Transcript** (✏️) - Opens edit mode for the transcript text
- **Summarize** (📝) - Calls summarize API to generate summary
- **Custom Prompt** (⚡) - Allows user to enter custom prompt for AI processing

### Button placement
- Place all 3 buttons in modal header, next to the close (X) button
- Order: [Edit] [Summarize] [Custom Prompt] [X]

### Button styling
- Icon buttons with emoji icons (✏️ 📝 ⚡)
- 14px size, consistent with ListenView button styling
- Subtle hover states, matching existing modal button styles

### Edit Transcript (✏️)
- **Behavior**: Inline editable text in the modal
- **Editable content**: Original text only
- **Controls**: Save/Cancel buttons appear when editing

### Summarize (📝)
- **Display**: Separate modal (new modal opens)
- **Content**: Full summary including:
  - TLDR (short summary)
  - Bullet points
  - Action items
- **Source text**: Both original and translated text

### Custom Prompt (⚡)
- **Input**: User selects from saved prompts (configured in Settings)
- **Display**: Separate modal (new modal opens)
- **Configuration**: Prompts managed in Settings view ( phase)
- **Source text**: Both original and translated text

</decisions>
