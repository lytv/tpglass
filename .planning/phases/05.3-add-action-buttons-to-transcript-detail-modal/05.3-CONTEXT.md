# Phase 05.3: Add Action Buttons to Transcript Detail Modal - Context

**Gathered:** 2026-03-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Add action buttons (Copy and Save) to the translation detail modal in SttView.js. The modal appears when user clicks on a translated transcript message. Currently it only shows text with a close button.

</domain>

<decisions>
## Implementation Decisions

### Action buttons
- Add Copy button to copy transcript to clipboard
- Add Save button to save transcript to .txt file
- Reuse existing IPC patterns from ListenView (handleCopy, handleSave)

### Button placement
- Place buttons in modal header, next to the close (X) button
- Order: [Copy] [Save] [X]

### Button styling
- Icons only (no text labels)
- 14px icons, consistent with ListenView button styling
- Subtle hover states, matching existing modal button styles

### Content scope
- Copy/Save includes both original AND translated text
- Format: "Original: [text]\n\nTranslation: [text]"
- Consistent with copy/save behavior from Phase 1

</decisions>

<specifics>
## Specific Ideas

- Reuse ListenView.js button patterns for consistency
- Follow existing modal styling in SttView.js
- IPC handlers can leverage featureBridge.js patterns already in place

</specifics>
