# Phase 05.3 Plan 02: Backend Summarize Service

## Summary

Created backend summarize service and IPC handlers to support the Summarize button in the transcript modal. This enables the frontend to call `window.api.summarize.summarize()` and receive structured summaries.

## Implementation Details

### summarizeService.js
- **Location**: `src/features/common/services/summarizeService.js`
- **Pattern**: Follows TranslationService pattern
- **Dependencies**: createLLM from ai/factory, modelStateService

### Core Functionality
- **Input**: { originalText, translatedText }
- **Output**: { success, tldr, bulletPoints[], actionItems[], error }
- **LLM**: Uses modelStateService to get API key and model info
- **Prompt**: Requests TLDR (1-2 sentences), Key Points (3-5), Action Items

### IPC Handler
- **Channel**: `summarize:summarize`
- **Location**: `src/bridge/featureBridge.js`
- **Routes**: Calls summarizeService.summarize() with originalText/translatedText

### Preload Bridge
- **API**: `window.api.summarize.summarize(originalText, translatedText)`
- **Location**: `src/preload.js`
- **Namespace**: `api.summarize`
