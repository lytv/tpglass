# Phase 05.4.1: Keyboard shortcut for checkbox selection - Research

**Researched:** 2026-03-13
**Domain:** Frontend keyboard navigation in LitElement components
**Confidence:** HIGH

## Summary

This phase adds keyboard navigation support for transcript checkbox selection in the Listen view (SttView component). The existing Phase 05.4 implemented hover-based checkbox visibility and multi-select functionality. This phase extends that to support keyboard-driven selection.

The app uses LitElement web components with a pattern of handling keyboard events either via `@keydown` template directives or `document.addEventListener` in `connectedCallback`. The SttView component already has `selectedTranscriptIds` state, checkbox rendering, and toggle methods that can be reused.

**Primary recommendation:** Implement keyboard selection mode as a new state property in SttView.js, use `@keydown` handler on the transcription container for Arrow/Space/Enter/Escape keys, and add CSS for visual focus indicator on the currently focused row.

### Pattern 1: Local Keyboard Handler in LitElement
**What:** Add `@keydown` handler to the main container element in the LitElement template
**When to Use:** When handling view-specific keyboard shortcuts that shouldn't interfere with other views
**Example:**
```javascript
// In SttView.js render() method
render() {
    return html`
        <div class="transcription-container"
            @keydown="${this._handleKeyboardNavigation}"
            tabindex="0">
            ...
        </div>
    `;
}
```

### Pattern 2: Selection Mode State
**What:** Boolean flag to track when keyboard selection mode is active
**When to Use:** When a mode needs to persist until explicitly exited
**Example:**
```javascript
// In SttView.js
static properties = {
    // ... existing properties
    isSelectionMode: { type: Boolean, state: true },
    keyboardFocusedIndex: { type: Number, state: true },
    selectionAnchor: { type: Number, state: true },  // For Shift+Arrow range selection
};

// In constructor

```

### Anti-Patterns to Avoid
- **Global shortcuts for view-specific actions:** This feature is scoped to Listen view only, so use local keyboard handling, not Electron globalShortcut
- **Ignoring focus management:** The container needs `tabindex="0"` to receive keyboard events, and focus should be managed visually with CSS

## Common Pitfalls

### Pitfall 1: Container Not Focusable
**What goes wrong:** Keyboard events don't fire on the container
**Why it happens:** `<div>` elements don't receive focus by default, need `tabindex="0"`
**How to avoid:** Add `tabindex="0"` to the transcription container, or use `@keydown` on the element with `.focus()`
**Warning signs:** Pressing keys does nothing, no console errors

### Pitfall 2: Checkbox Visibility Not Tied to Selection Mode
**What goes wrong:** Checkboxes only show on hover, but keyboard selection needs them visible
**Why it happens:** Existing CSS `.checkbox-wrapper { opacity: 0 }` only shows on hover
**How to avoid:** Add a class to container when `isSelectionMode` is true, CSS rule: `.selection-mode .checkbox-wrapper { opacity: 1 }`
**Warning signs:** Keyboard navigation works but checkboxes remain invisible

### Pitfall 3: Arrow Key Scrolling vs Navigation
**What goes wrong:** Arrow keys scroll the container instead of moving focus
**Why it happens:** Default browser behavior for overflow containers
**How to avoid:** Call `e.preventDefault()` in the keydown handler for Arrow keys
**Warning signs:** Focus moves but view scrolls away from focused item

### Pitfall 4: Range Selection Without Anchor
**What goes wrong:** Shift+Arrow doesn't maintain selection range
**Why it happens:** Missing anchor point tracking
**How to avoid:** Store `selectionAnchor` when a non-Shift selection happens, use it as start point for Shift+Arrow range
**Warning signs:** Shift+Arrow behaves same as regular Arrow

### Pitfall 5: Focus Lost on Checkbox Click
**What goes wrong:** Clicking checkbox removes focus from container, breaking keyboard navigation
**Why it happens:** Clicking checkbox focuses the input element
**How to avoid:** After checkbox click, programmatically restore focus to container: `container.focus()`
**Warning signs:** Have to click container again after checking a box to continue navigation

## Code Examples

### Example 1: Keyboard Navigation Handler
```javascript
// Add to SttView.js
_handleKeyboardNavigation(e) {
    const container = this.shadowRoot.querySelector('.transcription-container');
    const messages = this.sttMessages;

    switch(e.key) {
        case ' ':
            // Space: Toggle selection mode
            e.preventDefault();
            if (this.isSelectionMode) {
                this.isSelectionMode = false;
                this.keyboardFocusedIndex = -1;
            } else {
                this.isSelectionMode = true;
                this.keyboardFocusedIndex = 0;
            }
            break;

        case 'Escape':
            // Escape: Exit selection mode
            e.preventDefault();
            if (this.isSelectionMode) {
                this.isSelectionMode = false;
                this.keyboardFocusedIndex = -1;
            }
            break;

        case 'ArrowUp':
            e.preventDefault();
            if (this.keyboardFocusedIndex > 0) {
                this.keyboardFocusedIndex--;
            }
            break;

        case 'ArrowDown':
            e.preventDefault();
            if (this.keyboardFocusedIndex < messages.length - 1) {
                this.keyboardFocusedIndex++;
            }
            break;

        case ' ':
        case 'Enter':
            // Toggle selection for current row
            e.preventDefault();
            if (this.keyboardFocusedIndex >= 0 && this.keyboardFocusedIndex < messages.length) {
                const msgId = messages[this.keyboardFocusedIndex].id;
                this._toggleSelection(msgId);
                // Update anchor for range selection
                this.selectionAnchor = this.keyboardFocusedIndex;
            }
            break;

        case 'Shift':
            // Handle Shift+Arrow for range selection
            // (Implemented in Arrow key handlers above)
            break;
    }
}
```

### Example 3: Range Selection Logic
```javascript
// In ArrowDown/ArrowUp handlers, check for Shift key
_handleArrowDown(e) {
    if (e.shiftKey && this.selectionAnchor >= 0) {
        // Range selection: select all between anchor and new position
        const targetIndex = Math.min(this.keyboardFocusedIndex + 1, messages.length - 1);
        const start = Math.min(this.selectionAnchor, targetIndex);
        const end = Math.max(this.selectionAnchor, targetIndex);

        const idsToSelect = messages.slice(start, end + 1).map(m => m.id);
        const currentSelection = [...this.selectedTranscriptIds];

        // Add all in range to selection
        idsToSelect.forEach(id => {
            if (!currentSelection.includes(id)) {
                currentSelection.push(id);
            }
        });

        this.selectedTranscriptIds = currentSelection;
        this.keyboardFocusedIndex = targetIndex;
    } else {
        // Normal navigation
        this.keyboardFocusedIndex++;
        this.selectionAnchor = this.keyboardFocusedIndex;
    }
}
```
