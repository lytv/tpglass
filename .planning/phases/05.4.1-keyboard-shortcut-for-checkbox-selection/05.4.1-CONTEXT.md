# Phase 05.4.1: Keyboard shortcut for checkbox selection - Context

**Gathered:** 2026-03-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Add keyboard navigation support for transcript checkbox selection in Listen view. Users can enter selection mode, navigate between rows with arrow keys, toggle selection with Space/Enter, and use Shift+Arrow for range selection. Selection mode is local to Listen view.

</domain>

<decisions>
## Implementation Decisions

### Selection mode activation
- **Space** toggles selection mode on/off
- When active: checkboxes remain visible (no hover required)
- Selection mode persists until explicitly exited

### Selection mode exit
- **Escape** once: exits selection mode, keeps current selection
- **Escape** again: clears selection
- Alternative: clicking outside or closing modal also clears

### Navigation
- **Up/Down arrows**: navigate between transcript rows
- **Space/Enter**: toggle checkbox selection for current row
- Visual indicator shows which row is currently focused (keyboard cursor)

### Range selection
- **Shift+Arrow**: select all rows between current position and target
- Maintains anchor point for range operations

### Scope
- Shortcuts implemented in Listen view only (where transcript selection exists)
- Does not apply to other views with checkboxes

</decisions>

<specifics>
## Specific Ideas

- Follow existing keyboard handling patterns in SttView.js
- Reuse existing checkbox styling (already has .checkbox-wrapper and .transcript-checkbox)
- Visual focus indicator: add subtle highlight/border to current keyboard-selected row

</specifics>

## Existing Code Insights

### Reusable Assets
- SttView.js: Has checkbox rendering, _toggleSelection(), selectedTranscriptIds state, _toggleSelectAll()
- Existing checkbox CSS: .checkbox-wrapper with opacity transition
- ShortcutsService.js: Existing keybind pattern for reference

### Established Patterns
- Hover reveal for checkboxes (Phase 05.4)
- Modal overlay pattern for multi-select view
- Selection clears when modal closes

### Integration Points
- SttView: Add keyboard event handler for selection mode
- State: Add isSelectionMode boolean to track mode
- ListenView: May need to sync selection info display

</code_context>
