# Phase 05.3.1: Custom Prompts Settings - Context

**Gathered:** 2026-03-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Implement Custom Prompts Settings - allow users to create, edit, delete custom AI prompts in Settings, and execute them from the Custom Prompt modal in SttView.

This phase completes the Custom Prompt feature started in Phase 05.3 (placeholder modal → functional implementation).

</domain>

<decisions>
## Implementation Decisions

### Settings UI - Preset Management
- Add a new section below Translation language settings in SettingsView
- New section: "Custom Prompts" with list display
- List with inline actions: Create/Edit/Delete buttons next to each preset
- Expandable/collapsible section similar to existing preset section

### Custom Prompt Modal Flow (SttView)
- Dropdown to select saved prompt
- Run button to execute selected prompt
- Results display in the same modal (replaces placeholder)
- If no presets exist, show message: "No prompts configured. Create prompts in Settings."

### LLM Execution Handler
- Extend SummarizeService with runCustomPrompt() method
- Reuse existing LLM configuration (modelStateService)
- Reuse error handling and response parsing
- New IPC handler: 'custom-prompt:run'

### Default Prompts
- Provide 5 default prompts:
  1. "Extract Keywords" - Extract key terms and topics from the transcript
  2. "Sentiment Analysis" - Analyze the emotional tone of the transcript
  3. "Translate to [language]" - (Use target language from settings)
  4. "Summarize" - Short summary of content
  5. "Action Items" - Extract tasks and follow-ups
- Default prompts are read-only (is_default = 1)
- Users can create custom prompts (is_default = 0)

</decisions>
