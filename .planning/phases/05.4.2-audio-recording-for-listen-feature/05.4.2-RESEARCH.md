# Phase 05.4.2: Audio Recording for Listen Feature - Research

**Researched:** 2026-03-17
**Domain:** Audio recording, MP3 encoding, Electron IPC
**Confidence:** HIGH

## Summary

This phase implements automatic audio recording for the Listen feature. When user clicks Listen, the system will record both microphone and system audio, mix them together, and save as a single MP3 file when the user clicks Stop. The implementation requires creating a new RecordingService in the main process that interfaces with the lame encoder for MP3 encoding.

**Primary recommendation:** Create RecordingService in src/features/listen/recordingService.js that spawns lame as a child process, receives mixed PCM audio from listenCapture.js via IPC, encodes to MP3, and saves to the transcript folder with fallback to WAV on encoding failure.

## User Constraints (from CONTEXT.md)

### Implementation Decisions

### Audio Sources
- **Record both mic and system audio combined** — captures complete audio environment
- **Single mixed file** — one MP3 file with mic + system audio mixed together (not separate files)
- Mixing happens at the PCM level before encoding

### Audio Format
- **MP3 format** using lame encoder — smaller files, standard for voice recordings
- **Bitrate: 128 kbps** — good balance of quality and size
- If lame is not installed: **show clear error when user clicks Listen** (not silent skip)
- Cross-platform detection: check common paths on macOS (/opt/homebrew/bin/lame), Windows (PATH), Linux (PATH or /usr/bin/lame)
- If encoding fails mid-recording: **save partial recording as WAV** (don't lose data)

### Recording Trigger
- **Auto-start** — recording begins automatically when user clicks Listen
- **Auto-save on Stop** — when user clicks Stop to end Listen session, recording also stops and saves automatically
- No separate record/stop buttons needed — integrated with Listen workflow

### Save Location
- **Same folder as transcript** — MP3 saved to same directory where transcript .txt is saved
- **Filename pattern:** `recording-{YYYY-MM-DD-HHMM}.mp3` — matches transcript naming convention
- Uses same save path mechanism as transcript (remembers last used directory)

### Error Handling
- Check for lame encoder when user clicks Listen (not on app startup)
- If lame missing: show clear error message with installation instructions
- If encoding fails mid-session: fallback to WAV format for partial recording
- If save fails: show error toast, allow retry

### Specific Ideas

- Follow existing IPC patterns in preload.js (sendMicAudioContent, sendSystemAudioContent)
- Reuse file save flow from featureBridge.js (file:save-transcript handler)
- Design doc at docs/audio-recording.md outlines architecture: RecordingService (main process), lame encoder, ListenCapture sends PCM chunks

### Pattern 1: RecordingService (Main Process)
**What:** Manages lame child process for MP3 encoding
**When to use:** When encoding PCM audio to MP3 during Listen session
**Example:**
```javascript
// src/features/listen/recordingService.js
const { spawn } = require('child_process');
const fs = require('fs');
const path = require('path');

const SAMPLE_RATE = 24000;
const BITRATE = 128;

class RecordingService {
    constructor() {
        this.lamePath = null;
        this.tempWavPath = null;
        this.lameProcess = null;
    }

    // Cross-platform lame detection
    async findLame() {
        const { platform } = process;
        const possiblePaths = platform === 'darwin'
            ? ['/opt/homebrew/bin/lame', '/usr/local/bin/lame']
            : platform === 'win32'
                ? ['lame.exe']  // Rely on PATH
                : ['/usr/bin/lame', '/usr/local/bin/lame', 'lame'];  // Linux

        for (const p of possiblePaths) {
            try {
                // Check if executable exists
                await fs.promises.access(p.startsWith('/') ? p : p, fs.constants.X_OK);
                return p;
            } catch {
                continue;
            }
        }
        return null;
    }

    async checkLameAvailable() {
        if (this.lamePath) return true;
        this.lamePath = await this.findLame();
        return this.lamePath !== null;
    }

    async startRecording(sessionId) {
        const available = await this.checkLameAvailable();
        if (!available) {
            throw new Error('lame encoder not found. Please install: brew install lame (macOS) or sudo apt install lame (Linux)');
        }

        // Create temp file for WAV data (will be converted to MP3 on stop)
        const tempDir = require('os').tmpdir();
        this.tempWavPath = path.join(tempDir, `recording-${sessionId}-${Date.now()}.wav`);
        this.wavWriteStream = fs.createWriteStream(this.tempWavPath);

        // Write WAV header (will update later with actual size)
        this.writeWavHeader(this.wavWriteStream, 0);

        // Start lame process for streaming
        this.lameProcess = spawn(this.lamePath, [
            '-r',                    // Raw input
            '-s', SAMPLE_RATE.toString(),  // Sample rate
            '--bitwidth', '16',     // Bit width
            '--little-endian',      // Byte order
            '-b', BITRATE.toString(),       // Bitrate
            '-',                    // stdin
            '-'                     // stdout (we'll capture)
        ]);

        this.mp3Chunks = [];
        this.lameProcess.stdout.on('data', (data) => {
            this.mp3Chunks.push(data);
        });

        this.isRecording = true;
    }

    async writeAudioChunk(pcmData) {
        if (!this.isRecording) return;
        // PCM is already Int16, write directly
        this.wavWriteStream.write(Buffer.from(pcmData));
    }

    async stopAndSave(targetPath) {
        if (!this.isRecording) return;

        this.isRecording = false;

        // Close WAV stream
        this.wavWriteStream.end();

        // Wait for lame to finish encoding
        return new Promise((resolve, reject) => {
            this.lameProcess.on('close', async (code) => {
                try {
                    if (code === 0) {
                        // Combine chunks into final MP3
                        const mp3Buffer = Buffer.concat(this.mp3Chunks);
                        await fs.promises.writeFile(targetPath, mp3Buffer);
                        resolve({ success: true, path: targetPath });
                    } else {
                        // Encoding failed, save as WAV fallback
                        await fs.promises.copyFile(this.tempWavPath, targetPath.replace('.mp3', '.wav'));
                        resolve({ success: true, path: targetPath.replace('.mp3', '.wav'), fallback: true });
                    }
                } catch (err) {
                    reject(err);
                }

                // Cleanup temp file
                try {
                    await fs.promises.unlink(this.tempWavPath);
                } catch {}
            });

            this.lameProcess.stdin.end();
        });
    }

    writeWavHeader(stream, dataSize) {
        const buffer = Buffer.alloc(44);
        // RIFF header
        buffer.write('RIFF', 0);
        buffer.writeUInt32LE(36 + dataSize, 4);
        buffer.write('WAVE', 8);
        // fmt chunk
        buffer.write('fmt ', 12);
        buffer.writeUInt32LE(16, 16); // chunk size
        buffer.writeUInt16LE(1, 20);   // audio format (PCM)
        buffer.writeUInt16LE(1, 22);   // channels (mono)
        buffer.writeUInt32LE(SAMPLE_RATE, 24);
        buffer.writeUInt32LE(SAMPLE_RATE * 2, 28); // byte rate
        buffer.writeUInt16LE(2, 32);   // block align
        buffer.writeUInt16LE(16, 34);  // bits per sample
        // data chunk
        buffer.write('data', 36);
        buffer.writeUInt32LE(dataSize, 40);
        stream.write(buffer);
    }
}

module.exports = new RecordingService();
```

### Anti-Patterns to Avoid

- **Streaming directly to MP3 without temp file:** If app crashes mid-session, audio is lost. Use temp WAV file or buffer.
- **Checking for lame at app startup:** User may install lame after app starts. Check when Listen is clicked.
- **Blocking UI during encoding:** MP3 encoding should happen asynchronously after Stop is clicked.
- **Not handling missing lame gracefully:** Show clear error message with installation instructions.

## Common Pitfalls

### Pitfall 1: Lame Not Found on Different Platforms
**What goes wrong:** Lame installed in non-standard location, not in PATH
**Why it happens:** Different OS have different default paths
**How to avoid:** Check multiple possible paths, show clear error with installation instructions
**Warning signs:** "lame not found" error when clicking Listen

### Pitfall 2: Audio Desync Between Mic and System
**What goes wrong:** Mic and system audio arrive at different times, not synchronized in final recording
**Why it happens:** Two separate IPC channels with different latencies
**How to avoid:** Mix audio in renderer (listenCapture.js) before sending to RecordingService, or add timestamps and resync in main process
**Warning signs:** Audio seems offset in playback

### Pitfall 3: Large Memory Usage for Long Sessions
**What goes wrong:** Storing all MP3 chunks in memory for long recordings causes OOM
**Why it happens:** Buffering entire encoding result in memory
**How to avoid:** Write encoded chunks to temp file incrementally, or use streaming approach
**Warning signs:** Memory grows continuously during long Listen sessions

### Pitfall 4: Encoding Fails Mid-Session
**What goes wrong:** Lame crashes or errors during encoding, user loses entire recording
**Why it happens:** Corrupted audio data, lame bug, system resource issue
**How to avoid:** Always save partial WAV before attempting MP3 encoding, fallback to WAV on failure
**Warning signs:** "lame process exited with code X" errors

## Open Questions

1. **Audio Mixing Strategy**
   - What we know: Both mic and system audio are captured separately at 24kHz in listenCapture.js
   - What's unclear: Should mixing happen in renderer (JS) or main process (RecordingService)?
   - Recommendation: Mix in renderer to reduce IPC overhead - add combined stream in listenCapture.js

2. **WAV Header for Streaming**
   - What we know: WAV files need header with size, which we don't know until recording ends
   - What's unclear: How to handle streaming PCM to WAV without knowing final size?
   - Recommendation: Use raw PCM file, convert to WAV on stop, OR write placeholder header and update

3. **Session ID for Filename**
   - What we know: listenService has currentSessionId available
   - What's unclear: Should recording filename use session ID or timestamp?
   - Recommendation: Use timestamp pattern per spec: `recording-{YYYY-MM-DD-HHMM}.mp3`
