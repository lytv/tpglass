# Phase 05.2 Plan 01: Microphone Toggle Button Summary

## Objective

Add a microphone toggle button to the Listen view UI that allows users to disable microphone input to prevent acoustic leakage when they only want transcription from system audio.

## Completed Tasks

| Task | Name | Commit | Files |
|------|------|--------|-------|
| 1 | Add enableMic/disableMic functions to listenCapture.js | 2405579 | src/ui/listen/audioCore/listenCapture.js |
| 2 | Add mic toggle button to ListenView.js | 2e768cb | src/ui/listen/ListenView.js |
| 3 | Increase bar-controls width for new button | c230403 | src/ui/listen/ListenView.js |

## What Was Built

### Task 1: listenCapture.js Functions

- Added `micEnabled` state variable (default: true)
- Added `disableMic()` function that:
  - Stops all microphone MediaStream tracks
  - Disconnects and nullifies audioProcessor
  - Closes and nullifies audioContext
  - Logs "Microphone disabled" to console

- Added `enableMic()` function that:
  - Returns early if micMediaStream already exists
  - Re-initializes microphone capture using navigator.mediaDevices.getUserMedia
  - Sets up audio processing using platform-specific setupMicProcessing() (macOS/Windows) or setupLinuxMicProcessing() (Linux)
  - Logs "Microphone enabled" or "Failed to enable microphone" with error

- Added `isMicEnabled()` getter function
- Exposed functions via window.listenCapture

### Task 2: ListenView.js Mic Toggle Button

- Added `micEnabled` property to static properties
- Initialized `this.micEnabled = true` in constructor
- Added CSS styles for `.mic-toggle-button`:
  - Same styling as save-button/copy-button
  - `.mic-toggle-button:hover`: rgba(255, 255, 255, 0.15) background
  - `.mic-toggle-button.disabled`: color #ff5555 (red), opacity 0.8
  - `.mic-toggle-button.disabled:hover`: rgba(255, 85, 85, 0.2) background

- Added glass bypass for `.mic-toggle-button` classes
- Added `toggleMic()` method:
  - Toggles `this.micEnabled`
  - Calls `window.listenCapture.disableMic()` if disabled, `enableMic()` if enabled
  - Calls `this.requestUpdate()` to trigger re-render

- Added mic toggle button to render() in bar-controls (after Save button):
  - ON state: Standard microphone icon
  - OFF state: Microphone with strikethrough (diagonal line)
  - Button class: `mic-toggle-button ${!this.micEnabled ? 'disabled' : ''}`
  - Title: "Disable microphone" or "Enable microphone"
  - Click handler: `@click=${this.toggleMic}`

### Task 3: CSS Adjustment

- Increased `.bar-controls` width from 120px to 145px to accommodate the new button
