  <files>src/ui/listen/audioCore/listenCapture.js</files>
  <action>
Add two new functions to listenCapture.js for toggling microphone on/off:

1. Add a `micEnabled` state variable (default: true)
2. Add `disableMic()` function that:
   - Stops all microphone MediaStream tracks
   - Disconnects and nullifies audioProcessor
   - Closes and nullifies audioContext
   - Logs "Microphone disabled" to console

3. Add `enableMic()` function that:
   - Returns early if micMediaStream already exists (already enabled)
   - Re-initializes microphone capture using navigator.mediaDevices.getUserMedia with same settings as startCapture()
   - Sets up audio processing using platform-specific setupMicProcessing() (macOS/Windows) or setupLinuxMicProcessing() (Linux)
   - Logs "Microphone enabled" or "Failed to enable microphone" with error

4. Update window.listenCapture exports to include:
   - disableMic
   - enableMic
   - isMicEnabled (getter function returning micEnabled state)

Reference existing code patterns:
- Use existing SAMPLE_RATE constant (24000)
- Follow platform detection: isMacOS uses setupMicProcessing(), isLinux uses setupLinuxMicProcessing()
- Match existing getUserMedia options: { sampleRate, channelCount: 1, echoCancellation: true, noiseSuppression: true, autoGainControl: true }
