# Phase 05.2: Add Microphone Toggle Button to prevent acoustic leakage - Context

**Gathered:** 2026-03-09
**Status:** Ready for planning
**Source:** User PRD (Feature Request)

<domain>
## Phase Boundary

Add a microphone toggle button on the Listen view UI (next to the Save button) that allows users to turn off microphone input to prevent acoustic leakage when they only want transcription from system audio.

</domain>

<decisions>
## Implementation Decisions

### UI Placement
- Add toggle button in ListenView.js top-bar controls, next to the Save button
- Order: [Toggle View] [Translate] [Copy] [Save] [Mic Toggle]

### Visual Design
- Microphone icon with on/off states
- When OFF: show strikethrough or different color to indicate disabled state
- Tooltip displays current state
- Clear visual feedback when toggling

### Functionality
- Toggle microphone on/off
- When OFF: microphone audio stream is disabled/stopped
- When ON: microphone audio stream is active
- State should persist during session (or optionally remember user preference)

### Problem Being Solved
- Acoustic leakage: Sound from speakers being picked up by microphone
- AEC (Acoustic Echo Cancellation) not 100% effective
- No intelligent speaker diarization
- Duplicate/fragmented transcriptions appearing as both "Me" and "Them"

### Claude's Discretion
- Exact icon design and color scheme
- Whether to persist state to localStorage
- Exact tooltip text
- Animation/transition effects
- Keyboard shortcut (if any)
- How to handle the state when switching between views

</decisions>

<specifics>
## Specific Ideas

**Current system:**
- 2 audio sources in parallel: Microphone ("Me") + System Audio ("Them")
- AEC doesn't fully eliminate acoustic leakage
- Same voice appears twice: "Them" (complete) + "Me" (duplicate/fragment)

**Proposed solution:**
- Toggle button to disable microphone input
- User can turn off mic when only listening to system audio
- Prevents duplicate transcriptions

</specifics>
