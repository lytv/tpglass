# Phase 05.4: Multi-Select Transcript Detail Display - Research

**Researched:** 2026-03-12
**Domain:** LitElement UI, Electron desktop app, transcript management
**Confidence:** HIGH

## Summary

This phase adds multi-select capability to the transcript list view, allowing users to select multiple transcripts and view them side-by-side in a modal. The implementation requires checkbox selection with hover reveal, horizontal scrolling panels for display, and combined text handling for LLM operations (Summarize, Custom Prompt).

**Primary recommendation:** Extend SttView.js with new state properties for selection management, add checkbox UI with hover visibility, implement horizontal scrolling modal, and modify action handlers to work with combined text.

## User Constraints (from CONTEXT.md)

### Locked Decisions
- **Checkboxes + Select All**: Each transcript row has a checkbox; Select All button at top of list
- **Hover reveal**: Checkboxes and Select All appear on hover over the transcript list area
- **Select All scope**: Selects all transcripts in current session (all items in sttMessages)
- **Selection persistence**: Clears selection when modal closes
- **Side-by-side panels**: Each selected transcript in its own panel, panels arranged horizontally
- **Horizontal scroll**: When many transcripts selected, scroll horizontally to see more panels
- **Panel content**: Each panel shows both original + translated text (same as current single-view)
- **Selection mode**: When checkboxes visible (hover state), clicking a transcript opens single detail (not toggle selection)
- **Click with multi-select**: If multiple selected and user clicks one -> switch to single transcript view with that one selected
- **Combine text**: When multiple selected, Edit/Summarize/Custom Prompt buttons combine ALL selected transcripts into single text
- **Single execution**: Run Summarize/Custom Prompt once on combined text, not once per transcript
- **Results**: Display single combined result in the modal

### Deferred Ideas (OUT OF SCOPE)
- Batch export of multiple transcripts - future phase
- Individual results for each transcript in multi-select - future phase

### Pattern 1: Checkbox Selection with Hover Reveal

**What:** Checkboxes appear only on hover over transcript list area, Select All button appears on hover at top.

**When to use:** When implementing multi-select with clean default UI.

**Implementation approach:**
```javascript
// In static styles - add checkbox styles
.checkbox-wrapper {
    opacity: 0;
    transition: opacity 0.2s ease;
}

.transcription-container:hover .checkbox-wrapper {
    opacity: 1;
}

// New state properties
static properties = {
    // ... existing properties
    selectedTranscriptIds: { type: Array, state: true },
    isHoveringList: { type: Boolean, state: true },
};

// In constructor
this.selectedTranscriptIds = [];
this.isHoveringList = false;

// Handler methods
_toggleSelection(msgId) {
    const ids = [...this.selectedTranscriptIds];
    const idx = ids.indexOf(msgId);
    if (idx >= 0) {
        ids.splice(idx, 1);
    } else {
        ids.push(msgId);
    }
    this.selectedTranscriptIds = ids;
}

_toggleSelectAll() {
    if (this.selectedTranscriptIds.length === this.sttMessages.length) {
        this.selectedTranscriptIds = [];
    } else {
        this.selectedTranscriptIds = this.sttMessages.map(m => m.id);
    }
}
```

**Source:** LitElement documentation - reactive properties and event handling

### Pattern 3: Combined Text for LLM Operations

**What:** When multiple transcripts selected, combine all text into single string for Summarize/Custom Prompt operations.

**When to use:** When batch processing multiple transcripts with LLM APIs.

**Implementation approach:**
```javascript
_getCombinedText() {
    return this.selectedTranscriptIds
        .map(id => {
            const msg = this.sttMessages.find(m => m.id === id);
            if (!msg) return '';

            let text = `${msg.speaker}: ${msg.text}`;
            // Include translation if available
            if (this.showTranslation && this.translations[id]) {
                text += `\n  Translation: ${this.translations[id]}`;
            }
            return text;
        })
        .filter(t => t)
        .join('\n\n---\n\n');
}

// Updated _handleSummarize
async _handleSummarize() {
    const text = this.selectedTranscriptIds.length > 1
        ? this._getCombinedText()
        : this.expandedTranslation?.text || '';

    // ... rest of implementation
}
```

**Source:** Standard pattern for combining multiple text inputs - simple concatenation with separators

### Pattern 4: Modal State Management

**What:** Handle multi-panel modal state including selection clearing on close.

**When to use:** When modal needs to manage complex state with cleanup.

**Implementation approach:**
```javascript
// New state for multi-select modal
_multiSelectModalOpen: { type: Boolean, state: true },

// Clear selection when modal closes
_closeMultiSelectModal() {
    this._multiSelectModalOpen = false;
    // Clear selection as per requirement
    this.selectedTranscriptIds = [];
    this.expandedTranslation = null;
}

// Handle click with multi-select (switch to single view)
_handleTranscriptClick(msg) {
    if (this.selectedTranscriptIds.length > 0) {
        // Multi-select active - switch to single view
        this.selectedTranscriptIds = [msg.id];
        this.expandedTranslation = msg;
    } else {
        // Normal behavior - open single detail
        this._handleTranslationClick(msg);
    }
}
```

**Source:** Existing SttView.js patterns - modal state management

### Anti-Patterns to Avoid

- **Using native browser checkboxes without styling**: Custom styled checkboxes match dark theme better
- **Combining translations incorrectly**: Ensure each transcript's translation stays associated with its original
- **Not clearing selection on modal close**: Per requirements, selection must clear when modal closes
- **Running LLM operation per transcript**: Must run once on combined text, not iterate

## Common Pitfalls

### Pitfall 1: Checkbox Click Triggers Transcript Click

**What goes wrong:** Clicking checkbox also opens transcript modal due to event bubbling.

**Why it happens:** Both checkbox and transcript div have click handlers.

**How to avoid:** Use `@click.stop` on checkbox or separate clickable area from checkbox:
```javascript
// In template
<div class="message-wrapper ${wrapperClass}" @click="${() => this._handleTranscriptClick(msg)}">
    <input
        type="checkbox"
        class="checkbox-wrapper"
        .checked="${this.selectedTranscriptIds.includes(msg.id)}"
        @click="${(e) => e.stopPropagation()}"
        @change="${() => this._toggleSelection(msg.id)}"
    >
    ...
</div>
```

**Warning signs:** Modal opens immediately when clicking checkbox.

### Pitfall 2: Select All Includes Non-Final Messages

**What goes wrong:** Select All includes partial/incomplete transcripts.

**Why it happens:** sttMessages contains both partial and final messages.

**How to avoid:** Filter to only final messages in Select All:
```javascript
_toggleSelectAll() {
    const finalIds = this.sttMessages
        .filter(m => m.isFinal)
        .map(m => m.id);

    if (this.selectedTranscriptIds.length === finalIds.length) {
        this.selectedTranscriptIds = [];
    } else {
        this.selectedTranscriptIds = finalIds;
    }
}
```

**Warning signs:** Partial transcripts appear in multi-select modal.

### Pitfall 3: Translation Not Available for Combined Text

**What goes wrong:** Combined text shows original only, missing translations.

**Why it happens:** Translation lookup uses message ID, not combined index.

**How to avoid:** Map each selected ID to its translation:
```javascript
// In _getCombinedText()
if (this.showTranslation && this.translations[msg.id]) {
    text += `\n  Translation: ${this.translations[msg.id]}`;
}
```

**Warning signs:** Combined summary lacks translated content.

### Combined Text Handler

```javascript
_getTextForAction() {
    // Single transcript
    if (this.selectedTranscriptIds.length === 1) {
        const msgId = this.selectedTranscriptIds[0];
        const msg = this.sttMessages.find(m => m.id === msgId);
        let text = msg?.text || '';

        if (this.showTranslation && this.translations[msgId]) {
            text += `\n\nTranslation: ${this.translations[msgId]}`;
        }
        return text;
    }

    // Multiple transcripts - combine
    return this.selectedTranscriptIds
        .map(id => {
            const msg = this.sttMessages.find(m => m.id === id);
            if (!msg) return '';

            let text = `${msg.speaker}: ${msg.text}`;
            if (this.showTranslation && this.translations[id]) {
                text += `\n  Translation: ${this.translations[id]}`;
            }
            return text;
        })
        .filter(t => t)
        .join('\n\n---\n\n');
}
```

## Open Questions

1. **Should Select All include partial transcripts?**
   - What we know: sttMessages contains both partial (isPartial) and final (isFinal) messages
   - What's unclear: User intent for partial transcripts in multi-select
   - Recommendation: Only include final transcripts (isFinal === true) - partial content is incomplete

2. **Maximum number of panels?**
   - What we know: Horizontal scroll handles overflow
   - What's unclear: Performance with large number of selected transcripts
   - Recommendation: No hard limit, but consider showing count badge when > 5 selected

3. **Edit in multi-select mode?**
   - What we know: Single edit saves to sttMessages array
   - What's unclear: How to handle editing multiple at once
   - Recommendation: Edit first transcript only, or disable edit in multi-select mode
