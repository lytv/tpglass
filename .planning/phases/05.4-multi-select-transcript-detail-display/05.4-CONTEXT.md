# Phase 05.4: Multi-Select Transcript Detail Display - Context

**Gathered:** 2026-03-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Allow users to select multiple transcripts from the transcript list and view them together in a modal. When multiple selected, actions (Summarize, Custom Prompt) combine all selected text into one input. Single-select behavior remains as existing (click opens detail modal).

</domain>

<decisions>
## Implementation Decisions

### Selection mechanism
- **Checkboxes + Select All**: Each transcript row has a checkbox; Select All button at top of list
- **Hover reveal**: Checkboxes and Select All appear on hover over the transcript list area
- **Select All scope**: Selects all transcripts in current session (all items in sttMessages)
- **Selection persistence**: Clears selection when modal closes

### Display layout
- **Side-by-side panels**: Each selected transcript displays in its own panel, panels arranged horizontally
- **Horizontal scroll**: When many transcripts selected, scroll horizontally to see more panels
- **Panel content**: Each panel shows both original + translated text (same as current single-view)

### Click behavior
- **Selection mode**: When checkboxes visible (hover state), clicking a transcript opens single detail (not toggle selection)
- **Click with multi-select**: If multiple selected and user clicks one → switch to single transcript view with that one selected

### Bulk actions
- **Combine text**: When multiple selected, Edit/Summarize/Custom Prompt buttons combine ALL selected transcripts into single text
- **Single execution**: Run Summarize/Custom Prompt once on combined text, not once per transcript
- **Results**: Display single combined result in the modal

</decisions>

<specifics>
## Specific Ideas

- Keep existing modal structure and styling (translation-modal, summary-modal)
- Action buttons (Edit, Summarize, Custom Prompt) remain in modal header
- Selection state stored in SttView (new state: selectedTranscriptIds array)

</specifics>

## Existing Code Insights

### Reusable Assets
- SttView.js: Has sttMessages array, expandedTranslation state, modal rendering
- Modal components: translation-modal-overlay, summary-modal already exist
- Action button pattern: Edit/Summarize/Custom Prompt buttons already in modal header

### Established Patterns
- Modal overlay pattern (click outside to close)
- Emoji icon buttons (✏️ 📝 ⚡)
- Translation toggle and display (original + translated)

### Integration Points
- SttView: Add selectedTranscriptIds state, checkbox rendering in transcript list
- Modal: Support multiple panels in side-by-side layout
- SummarizeService: May need combined-text input handling

</code_context>
