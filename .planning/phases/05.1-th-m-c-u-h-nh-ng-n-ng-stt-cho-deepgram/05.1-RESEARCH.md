# Phase 05.1: Thêm cấu hình ngôn ngữ STT cho Deepgram - Research

**Researched:** 2026-03-08
**Domain:** Electron desktop app, speech-to-text configuration, settings persistence
**Confidence:** HIGH

## Summary

This phase adds user-configurable Deepgram STT language settings that persist across sessions. The implementation follows the existing translation settings pattern already established in the codebase. The key integration points are: (1) add `sttLanguage` to electron-store defaults, (2) create getSttLanguage/setSttLanguage IPC handlers in settingsService, (3) expose through preload bridge, (4) read stored language in sttService when initializing Deepgram sessions.

**Primary recommendation:** Mirror the translation settings implementation pattern exactly - reuse the existing getTranslationSettings/setTranslationSettings pattern, use electron-store with the same user-scoped storage, and follow the same IPC handler registration in featureBridge.js.

### Pattern 1: Settings Service Handler Pattern
**What:** Adding new settings getters/setters to settingsService.js
**When to use:** For any new persistent user preference
**Example:**
```javascript
// Add to electron-store defaults (line 11-19)
const store = new Store({
    name: 'pickle-glass-settings',
    defaults: {
        // ... existing defaults
        sttLanguage: 'en-US'  // NEW
    }
});

// Add new handler functions (after line 456)
async function getSttLanguage() {
    try {
        const settings = await getSettings();
        return settings.sttLanguage || 'en-US';
    } catch (error) {
        console.error('[SettingsService] Error getting STT language:', error);
        return 'en-US';
    }
}

async function setSttLanguage(language) {
    try {
        const settings = await getSettings();
        settings.sttLanguage = language;
        await saveSettings(settings);

        // Notify windows of STT language change
        windowNotificationManager.notifyRelevantWindows('stt-language-updated', { language });

        return { success: true, language };
    } catch (error) {
        console.error('[SettingsService] Error setting STT language:', error);
        return { success: false, error: error.message };
    }
}
```

### Pattern 2: IPC Handler Registration
**What:** Registering IPC handlers in featureBridge.js
**When to use:** To expose settings to renderer process
**Example:**


### Pattern 3: Preload API Exposure
**What:** Exposing IPC methods to renderer via preload bridge
**When to use:** To allow SettingsView.js to call main process
**Example:**
```javascript
// Add in preload.js translation section (around line 322)
stt: {
    getLanguage: () => ipcRenderer.invoke('settings:getSttLanguage'),
    setLanguage: (language) => ipcRenderer.invoke('settings:setSttLanguage', { language }),
    onSttLanguageUpdated: (callback) => ipcRenderer.on('stt-language-updated', callback),
    removeOnSttLanguageUpdated: (callback) => ipcRenderer.removeListener('stt-language-updated', callback)
}
```

### Pattern 4: STT Service Integration
**What:** Reading stored language when initializing Deepgram sessions
**When to use:** To use user-configured language instead of hardcoded value
**Example:**
```javascript
// In sttService.js initializeSttSessions (around line 152)
// Current code:
async initializeSttSessions(language = 'en') {
    const effectiveLanguage = process.env.OPENAI_TRANSCRIBE_LANG || language || 'en';
// Change to:
async initializeSttSessions(language = 'en') {
    // Try to get stored STT language from settings
    let storedLanguage = 'en-US';
    try {
        const settings = await settingsService.getSttLanguage?.();
        if (settings?.language) storedLanguage = settings.language;
    } catch (e) {
        console.log('[SttService] Using default language - settings not available');
    }
    const effectiveLanguage = process.env.OPENAI_TRANSCRIBE_LANG || language || storedLanguage || 'en-US';
```

### Anti-Patterns to Avoid

- **Creating a separate store:** Use existing electron-store instance in settingsService.js
- **Hardcoding language in deepgram.js:** Always pass language from sttService, never hardcode
- **Not broadcasting settings changes:** Always notify relevant windows when language changes
- **Ignoring the settings load race condition:** Note from recent fix (commit 29ab0b7) that settings loading can have race conditions - use dynamic reloading pattern

## Common Pitfalls

### Pitfall 1: Language Code Format Mismatch
**What goes wrong:** Deepgram expects BCP-47 format (e.g., 'en-US') but translation uses ISO-639-1 (e.g., 'en')
**Why it happens:** Deepgram's API uses locale-specific codes; translation service uses simple language codes
**How to avoid:** Use 'en-US' as default and store full locale codes; verify Deepgram supports the selected language
**Warning signs:** "Invalid language" errors from Deepgram API, transcription not starting

### Pitfall 2: Settings Not Loaded Before STT Initialization
**What goes wrong:** sttService tries to read settings before they're initialized
**Why it happens:** Race condition between app startup and settings loading
**How to avoid:** Use try/catch with fallback to default; similar fix applied in commit 29ab0b7 for translation settings
**Warning signs:** Language always defaults to 'en-US' regardless of stored preference

### Pitfall 3: Not Broadcasting Settings Changes
**What goes wrong:** Other windows not aware of language change
**Why it happens:** Forgetting to call windowNotificationManager after saving
**How to avoid:** Always call notifyRelevantWindows with 'stt-language-updated' event
**Warning signs:** Language doesn't update in active Listen sessions

### Existing Translation Settings Pattern (settingsService.js:426-456)
```javascript
// Translation settings handlers
async function getTranslationSettings() {
    try {
        const settings = await getSettings();
        return {
            enabled: settings.translationEnabled || false,
            language: settings.translationLanguage || 'en'
        };
    } catch (error) {
        console.error('[SettingsService] Error getting translation settings:', error);
        return { enabled: false, language: 'en' };
    }
}

async function setTranslationSettings({ enabled, language }) {
    try {
        const settings = await getSettings();
        settings.translationEnabled = enabled;
        settings.translationLanguage = language;

        await saveSettings(settings);

        // Notify windows of translation settings change
        windowNotificationManager.notifyRelevantWindows('translation-settings-updated', { enabled, language });

        return { success: true, enabled, language };
    } catch (error) {
        console.error('[SettingsService] Error setting translation settings:', error);
        return { success: false, error: error.message };
    }
}
```
