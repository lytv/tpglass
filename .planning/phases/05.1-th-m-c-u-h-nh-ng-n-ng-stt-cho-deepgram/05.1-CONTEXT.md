# Phase 05.1: Thêm cấu hình ngôn ngữ STT cho Deepgram - Context

**Gathered:** 2026-03-08
**Status:** Ready for planning

<domain>
## Phase Boundary

Add user-configurable language settings for Deepgram speech-to-text service. Users can select their preferred STT language, which persists across sessions. This does NOT include auto-detection during transcription — only a user-selected default.

**Out of scope:**
- Auto-detection of language during transcription
- Per-session language override (always use stored preference)

</domain>

<decisions>
## Implementation Decisions

### Default language behavior
- **Remember last choice**: Store user's last selected language in electron-store
- **First-time default**: If no stored preference exists, default to `en-US`
- **Storage location**: Use electron-store (same as translation settings)

### Persistence
- Store in electron-store under `sttLanguage` key
- Pattern follows existing `translationEnabled`/`translationLanguage` settings
- Settings load via `settingsService.getSettings()` and `saveSettings()`

</decisions>

<specifics>
## Specific Ideas

- "Remember the last language the user selected" — similar to how translation language persists
- Use electron-store like translation settings do — consistent with existing pattern

</specifics>

<codebase_context>

</codebase_context>
