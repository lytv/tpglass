## Phase Details

### Phase 1: Save Transcript to File

**Goal:** Users can export their speech-to-text transcripts to local .txt files

**Depends on:** Nothing (first phase)

**Requirements:** FILE-01, FILE-02, FILE-03, FILE-04, FILE-05, FILE-06

**Success Criteria** (what must be TRUE):

1. User can click a Save button in the Listen view UI to initiate transcript save
2. Native OS save dialog opens allowing user to select file location
3. Default filename is pre-filled with timestamp format (transcript-{timestamp}.txt)
4. Transcript text content is written to selected .txt file
5. User receives visible confirmation message after successful save
6. User receives visible error message with reason if save operation fails
